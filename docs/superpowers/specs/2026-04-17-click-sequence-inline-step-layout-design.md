# 点击步骤编辑器行内布局设计

## 背景

当前实验点击序列的每个步骤字段使用竖排 `label + input/select`，单步会占用较高纵向空间。按钮设置项较多时，编辑区域滚动频繁，影响批量配置效率。

## 目标

- 让点击步骤编辑器中的单个设置项在桌面端以“左侧标签、右侧输入控件”同一行显示
- 优先压缩 `selector`、`设值`、数值类字段的纵向空间
- 仅调整点击步骤编辑器局部布局，不改变字段含义、顺序和保存逻辑

## 非目标

- 不修改实验点击序列运行时行为
- 不调整其他设置卡片的通用表单布局
- 不新增或删除字段

## 方案

在 `src/App.vue` 的点击步骤编辑器内为每个字段增加专用行类，保持现有字段顺序不变。`选择器` 与 `设值` 这类长输入保持整行宽度，其余字段在桌面端按两列排列；每个字段内部改为标签固定宽度、输入控件自适应剩余空间。窄屏下自动退回单列，避免挤压输入区域。

## 测试

- 为设置页布局测试增加行内字段类名断言，锁定模板结构
- 运行 `tests/settings-app-layout.test.ts`
- 运行 `npm run build` 确认样式改动不影响打包
