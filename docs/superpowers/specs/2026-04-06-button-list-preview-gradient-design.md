# 按钮列表承接当前预览视觉设计

## 背景

当前设置页右侧存在“当前预览”卡片，用于展示当前选中按钮的图标、标题和位置/动作类型信息。该卡片使用了一层轻微的渐变底色来强调“当前正在编辑的按钮”。

本次调整的目标是移除右侧“当前预览”卡片，并将这层视觉强调迁移到左侧按钮列表，使左侧当前选中项直接承担原有的“当前预览”视觉职责。

## 目标

- 删除右侧设置区中的“当前预览”卡片
- 将原 `.preview-card` 的渐变底色语言迁移到左侧按钮列表当前选中项
- 保持现有按钮列表的排序、显隐、删除、选中行为不变
- 保持右侧表单编辑逻辑不变

## 非目标

- 不调整按钮列表的字段内容
- 不新增按钮列表的额外交互
- 不修改未选中列表项的视觉样式
- 不改动位置预览区域的结构和交互

## 设计决策

### 1. 删除“当前预览”结构

移除 `selectedItem` 条件块中的 `editor-card--preview` 区段，不再在右侧单独渲染当前按钮的摘要卡片。

理由：

- 右侧当前预览与左侧当前选中态表达的是同一对象，存在信息重复
- 删除该卡片可以释放右侧垂直空间，让表单编辑更直接

### 2. 将渐变底色迁移到左侧激活项

把原 `.preview-card` 的渐变背景迁移到 `.button-list__item.is-active`，使当前选中项继承原本“当前预览”的视觉焦点。

样式策略：

- 保留现有 active 边框高亮语义
- 在 active 态上叠加原预览卡片的线性渐变底色
- 背景仍保持与当前主题变量兼容，不引入固定色值

理由：

- “当前预览”的语义本质上就是“当前选中项”
- 将强调放回列表本身，可以减少视线往返

### 3. 不扩散到整个列表

仅当前选中项带渐变，不给所有列表项加渐变背景。

理由：

- 与已确认的视觉方向一致
- 避免列表信息密度下降
- 保持当前项仍然是唯一明确焦点

## 影响范围

### 模板

- `src/App.vue`

需要删除右侧“当前预览”卡片的模板结构。

### 样式

- `src/index.scss`

需要：

- 删除不再使用的 `.preview-card` 相关样式
- 调整 `.button-list__item.is-active` 以承接渐变背景
- 清理仅被该预览卡片引用、且不再复用的样式选择器

## 数据与行为影响

本次改动仅涉及界面结构与样式迁移，不涉及：

- 配置数据结构
- `selectedItem` 的选中逻辑
- 表单字段读写
- 位置预览数据

因此不需要配置迁移，也不预期产生运行时行为变化。

## 测试策略

本次优先进行现有测试回归与手工验证。

自动验证：

- 运行现有 Vitest 测试，确认模板删除和样式迁移未引入回归

手工验证：

- 选中任意按钮后，左侧当前项出现渐变底色
- 切换选中项后，渐变焦点随之切换
- 右侧不再显示“当前预览”卡片
- 排序、显隐、删除、右侧表单编辑行为保持正常

## 风险与控制

风险：

- 删除模板后，可能遗留未使用样式选择器
- active 态叠加渐变后，边框对比度可能不足

控制方式：

- 清理仅由预览卡片使用的样式
- 保留现有 active 边框与阴影逻辑，只迁移背景层

## 实施边界

本次实现限定为一次小范围 UI 收敛：

- 仅移除一个冗余卡片
- 仅迁移一处视觉强调
- 不顺带做其他列表样式重构
