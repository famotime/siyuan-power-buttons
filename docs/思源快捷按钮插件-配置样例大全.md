# 思源快捷按钮插件配置样例大全

- 文档日期：2026-04-15
- 适用版本：当前仓库实现基线，插件 `minAppVersion = 3.5.7`

这份文档是对 [思源快捷按钮插件-动作类型使用指南](/home/quincyzou/projects/siyuan-power-buttons/docs/思源快捷按钮插件-动作类型使用指南.md) 的补充，目标只有一个：给出可以直接照着修改的 JSON 样例。

适合场景：

- 你想批量配置，而不是在设置页一点点填
- 你想开启实验动作，但当前界面没有明显的实验开关入口
- 你想在多台设备之间迁移配置

## 1. 如何使用这些样例

推荐流程：

1. 先在插件设置中导出当前配置
2. 复制本文的样例片段，替换到导出的 JSON 里
3. 根据自己的按钮标题、图标、位置、动作参数做微调
4. 再导入配置文件

建议：

- 修改前先保留一份原始导出文件
- 每次只新增或修改 1 到 2 个按钮，便于排查问题
- 实验动作改完后，优先在桌面端回归测试

## 3. 顶层字段说明

| 字段 | 说明 |
| --- | --- |
| `version` | 当前配置版本，保持 `2` |
| `desktopOnly` | 当前默认是 `true`，表示主要面向桌面端 |
| `items` | 所有自定义按钮配置 |
| `disabledNativeButtons` | 被隐藏的原生按钮列表 |
| `experimental` | 实验能力总开关集合 |

建议：

- 如果只是改按钮，优先只动 `items`
- 如果实验动作无法执行，再检查 `experimental`

## 4. `items` 数组中的通用字段

每个按钮至少会包含以下字段：

```json
{
  "id": "your-button-id",
  "title": "按钮标题",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconInfo",
  "surface": "topbar",
  "order": 0,
  "actionType": "builtin-global-command",
  "actionId": "globalSearch",
  "tooltip": "提示文字"
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `id` | 按钮唯一标识，建议手工保持不重复 |
| `title` | 列表里显示的按钮名称 |
| `visible` | 是否显示 |
| `iconType` | 图标类型，常见为 `builtin`、`emoji`、`svg` |
| `iconValue` | 图标内容，取决于 `iconType` |
| `surface` | 显示位置 |
| `order` | 同一区域内排序，数字越小越靠前 |
| `actionType` | 动作类型 |
| `actionId` | 动作主标识，不同动作类型含义不同 |
| `tooltip` | 鼠标悬停提示 |

## 5. 可用位置 `surface` 样例

当前代码模型中的可配置位置，常用值如下：

```json
[
  "topbar",
  "statusbar-left",
  "statusbar-right",
  "canvas"
]
```

说明：

- 当前设置页主配置区域主要围绕以上位置工作
- 代码模型里还有 Dock 相关类型，但常规配置时优先使用上面四个值

## 6. 稳定动作样例

### 6.1 内置命令：全局搜索

```json
{
  "id": "builtin-search",
  "title": "搜索",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconSearch",
  "surface": "topbar",
  "order": 0,
  "actionType": "builtin-global-command",
  "actionId": "globalSearch",
  "tooltip": "打开全局搜索"
}
```

### 6.2 内置命令：大纲

```json
{
  "id": "builtin-outline",
  "title": "大纲",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconList",
  "surface": "statusbar-left",
  "order": 1,
  "actionType": "builtin-global-command",
  "actionId": "outline",
  "tooltip": "显示大纲"
}
```

### 6.3 插件命令：打开本插件设置

当前内部插件命令的 `actionId` 采用 `providerId:commandId` 格式。

```json
{
  "id": "plugin-open-settings",
  "title": "快捷按钮设置",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconSettings",
  "surface": "topbar",
  "order": 2,
  "actionType": "plugin-command",
  "actionId": "siyuan-power-buttons:open-settings",
  "tooltip": "打开思源快捷按钮设置"
}
```

### 6.4 插件命令：外部插件命令占位示例

如果另一个插件公开了命令，例如 provider id 是 `siyuan-doc-assist`，命令 id 是 `insert-doc-summary`，则写法类似：

```json
{
  "id": "plugin-doc-summary",
  "title": "插入摘要",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconSparkles",
  "surface": "topbar",
  "order": 3,
  "actionType": "plugin-command",
  "actionId": "siyuan-doc-assist:insert-doc-summary",
  "tooltip": "调用文档助手插入摘要"
}
```

前提：

- 对方插件已经实现 provider 协议
- 该命令能在当前插件设置页中被刷新到

## 7. 实验快捷键适配样例

### 7.1 编辑器加粗

```json
{
  "id": "exp-bold",
  "title": "加粗",
  "visible": true,
  "iconType": "builtin",
  "iconValue": "iconBold",
  "surface": "topbar",
  "order": 10,
  "actionType": "experimental-shortcut",

    "shortcut": "Ctrl+B",
    "sendEscapeBefore": false,
    "dispatchTarget": "active-editor",
    "allowDirectWindowDispatch": false
  }
}
```

适用：

- 编辑器内文字格式化

### 7.2 命令面板或全局搜索类快捷键

    "shortcut": "Ctrl+P",

    "dispatchTarget": "auto",
    "allowDirectWindowDispatch": false

}

适用：

- 页面级快捷键

## 12. 修改时最容易踩的坑

- `id` 重复：会让按钮难以区分，建议手工保持唯一
- `order` 混乱：建议同一位置从 `0` 开始递增
- `actionId` 和实验配置不一致：例如快捷键改了，但 `actionId` 没同步
- 文本选择器过于依赖中文文案：切换语言后容易失效
- CSS 选择器过宽：容易点到第一个匹配项，而不是你真正想点的元素
- 忘记开启 `experimental.shortcutAdapter` 或 `experimental.clickSequenceAdapter`

## 13. 推荐做法

- 先从完整配置骨架开始改，而不是从空 JSON 手写
- 每次只验证一个实验按钮
- 快捷键优先尝试 `dispatchTarget = "auto"` 或 `"active-editor"`
- 点击序列优先使用稳定标识，如 `id`、`data-id`
- 调整点击序列时，优先改延迟和超时，不要一开始就堆很多重试
