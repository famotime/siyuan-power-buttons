# 默认快捷按钮配置说明

## 默认配置定义在哪里

默认快捷按钮配置定义在 [src/core/config/defaults.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/defaults.ts:78) 的 `createDefaultConfig()`。

这个函数返回插件启动时使用的完整默认配置，包含：

- `version`
- `desktopOnly`
- `items`
- `disabledNativeButtons`
- `experimental`

其中：

- 默认按钮列表定义在同文件的 `items` 数组中
- 当前默认不再预置任何被禁用的原生按钮，因此 `disabledNativeButtons` 默认为空数组

当前默认按钮顺序为：

1. 今日日记
2. 最近文档
3. 数据历史
4. 集市
5. 重启所有插件
6. 切换到英文
7. 切换到中文
8. 仅导出当前文档
9. 随心按设置

## 默认配置在运行时如何生效

[src/core/config/store.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/store.ts:10) 中的 `ConfigStore` 会直接使用 `createDefaultConfig()`：

- 初始化内存配置时使用一次
- 用户执行“恢复默认配置”时再次使用

加载已保存配置时，会经过 [src/core/config/sanitize.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/sanitize.ts:174) 做清洗和补默认值；如果本地没有保存配置，最终就会回到 `createDefaultConfig()` 提供的默认结构。

## 文档示例配置在哪里

仓库内用于展示和导出的示例 JSON 在 [docs/siyuan-power-buttons-config.json](/D:/MyCodingProjects/siyuan-power-buttons/docs/siyuan-power-buttons-config.json:1)。

这个文件当前与 `createDefaultConfig()` 保持一致，适合作为：

- 文档示例
- 手动导入参考
- 回归核对样本

## 修改默认配置时该改哪些地方

如果你要调整“插件首次安装/恢复默认”时出现的按钮，请至少同步检查这些位置：

1. [src/core/config/defaults.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/defaults.ts:78)
2. [docs/siyuan-power-buttons-config.json](/D:/MyCodingProjects/siyuan-power-buttons/docs/siyuan-power-buttons-config.json:1)
3. [tests/config-store.test.ts](/D:/MyCodingProjects/siyuan-power-buttons/tests/config-store.test.ts:8)

建议流程：

1. 先修改 `createDefaultConfig()` 里的默认数据
2. 再同步更新 `docs/siyuan-power-buttons-config.json`
3. 最后更新对应测试断言，确保标题、数量、动作类型和实验开关都一致

## 新增一个默认按钮的最小做法

在 [src/core/config/defaults.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/defaults.ts:79) 的 `normalizeItemOrder([...])` 里新增一个 `createButtonItem({...})` 即可，例如：

```ts
createButtonItem({
  id: 'pb-example-button-id',
  title: '我的新按钮',
  iconType: 'iconpark',
  iconValue: 'iconpark:Plus',
  surface: 'statusbar-right',
  actionType: 'builtin-global-command',
  actionId: 'dailyNote',
  tooltip: '',
})
```

注意：

- `id` 需要唯一，建议沿用 `pb-...` 风格
- `surface` 必须是项目支持的可配置位置
- `actionType` 和 `actionId` 必须匹配
- 如果是 `experimental-shortcut` 或 `experimental-click-sequence`，还要补对应实验配置

## 哪些改动只是文档，不会影响真实默认值

只修改下面这些文件，不会改变插件实际默认配置：

- `docs/*.md`
- `docs/siyuan-power-buttons-config.json`

真正影响默认值的源头仍然是：

- [src/core/config/defaults.ts](/D:/MyCodingProjects/siyuan-power-buttons/src/core/config/defaults.ts:78)

## 验证方式

修改默认配置后，至少执行：

```bash
npm test -- tests/config-store.test.ts
```

如果要手动验证：

1. 启动插件
2. 清空该插件已保存配置，或执行“恢复默认配置”
3. 打开设置页，检查按钮标题、顺序、位置和动作是否符合预期
